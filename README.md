## Verification

Use Node.js 22 or later:

```bash
npm ci
npm run check:docs
npm run check:portfolio-authority
npm run check:deployment-policy
npm run test:deployment-selector
npm run lint:apps
npm run typecheck:all
npm run test:all
npm run build:all
npm run verify
npm run verify:release
```

The former GitHub Pages workflow is a build-only **Portfolio Vercel Readiness** workflow. It validates static outputs but does not deploy them. The guarded path-aware Vercel workflow remains the repository deployment controller; native Vercel Git fan-out must not be disabled until its credentials, activation variable, and rollback path are verified. See [Verification](docs/verification.md) and the [Vercel Consolidation Runbook](docs/VERCEL_CLEANUP_RUNBOOK.md).

## Vercel-Only Deployment Boundary

- Keep one isolated Vercel project per maintained deployed application.
- Keep GitHub as the editable source of truth; Vercel deployments are derivative outputs.
- Use the existing affected-project selector for controlled deployments.
- Treat documentation-only changes as non-deploying in the repository controller.
- Do not disable native Vercel Git deployments until the guarded workflow has been credentialed and validated with a single-project preview.
- Treat legacy non-Vercel hosts as historical evidence only.

## Responsible Scope

- Local-first does not mean encrypted, durable, or compliance-certified storage.
- Deterministic scores and recommendations are product heuristics, not certified predictions or professional advice.
- AI-assisted output requires review and should not be represented as independently verified.
- Medical, legal, financial, fraud, infrastructure, and safety-critical decisions remain outside project scope unless a project explicitly documents a narrower educational use.
