# Governance Summary

The portfolio uses four separate records for four separate purposes:

1. `config/portfolio-authority.json` — canonical public identity, source authority, deployment identity, tier, audience, and evidence boundary.
2. `config/vercel-projects.json` — deployment-selection and affected-project configuration.
3. Current documentation — human-readable status and review paths.
4. Dated audits — immutable historical observations.

The Portfolio Authority workflow guards structural consistency. Human review remains responsible for nuanced claims, live behavior, security interpretation, external validation, and destructive account actions.
