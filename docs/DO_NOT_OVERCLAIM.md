# Evidence-Safe Claim Rules

Use the strongest claim supported by current evidence—no stronger.

- Reachable URL → **deployed**, not fully verified.
- Passing build → **build-verified**, not production-ready.
- Passing automated tests → **tested within stated scope**, not bug-free.
- Local-first architecture → describe exact storage and network behavior.
- Packaged product → **commercially packaged**, not market-validated.
- Five-person pilot → report the exact sample and results, not broad user demand.
- Public source → **source-visible** when the license is not open source.
- Security review → state methods, date, findings, and unresolved risks.
- AI-assisted work → identify direction, decisions, review, and verification without implying every line was manually authored.

Correct or remove any statement that cannot be traced to source, test, deployment, transaction, participant record, or other credible evidence.
