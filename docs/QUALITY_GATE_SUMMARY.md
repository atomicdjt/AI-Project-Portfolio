# Portfolio Quality Gates

The portfolio uses the following evidence layers:

- repository linting, type checking, tests, and builds;
- project-specific browser and packaging checks where configured;
- canonical authority validation for source, deployment, and public-status consistency;
- human review for nuanced claims, security, accessibility, usability, and account-level actions;
- external validation protocol for real participant evidence.

No single gate establishes complete correctness, security, production readiness, or market demand.
