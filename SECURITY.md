# Security Policy

This repository is a public portfolio for review. It contains local-first browser applications, static demos, documentation, screenshots, and case studies.

## Reporting

If you find a security or privacy issue, report it privately to the repository owner through GitHub instead of posting sensitive examples publicly. Use synthetic samples whenever possible.

Do not include real personal data, credentials, financial records, medical records, legal documents, or other sensitive material in public issues, pull requests, screenshots, or reproduction steps.

## Scope

In scope:

- Portfolio application source code in `apps/`
- Public static deployments linked from the repository
- Documentation that could misrepresent privacy, safety, or security behavior

Out of scope:

- Third-party services outside this repository's control
- Dependency vulnerabilities without a reachable impact in the portfolio apps
- Social engineering, spam, denial-of-service testing, or intrusive scanning

## Expectations

These projects are portfolio demonstrations, not regulated production services. Security notes and threat-model claims should stay conservative, specific, and tied to verified behavior.
