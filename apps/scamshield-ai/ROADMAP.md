# ScamShield AI Roadmap

## Phase 1: Local-First MVP

- Static React application
- Deterministic local rule-based analysis
- Transparent risk scoring and explanations
- Evidence packet PDF
- Five synthetic demo cases
- Caregiver and plain-language modes
- Editable timeline and action checklist
- U.S. official reporting-resource configuration
- Netlify-ready static deployment

## Phase 2: Evidence and Accessibility Improvements

- Client-side OCR with clear confidence and correction workflows
- PDF text extraction without server upload
- Better URL normalization and domain-comparison logic
- Maintained verified reporting-resource workflow
- Spanish and additional plain-language translations
- Partner feedback with aging, disability, financial-counseling, and community organizations
- Formal WCAG 2.2 AA audit and remediation
- Broader legitimate-message false-positive dataset

## Phase 3: Optional Privacy-Preserving Intelligence

- Optional AI explanation layer, disabled by default
- On-device or local-model support where feasible
- Clear model limitations, provenance, and no-final-determination controls
- Institutional nonprofit version with configurable resources and templates
- Versioned evaluation dataset and published model/rule cards
- Grant-funded pilot with formal research and privacy governance

## Phase 4: Verified Integrations and Companion Products

- Integrations with verified reporting-resource directories
- Browser extension that captures text without opening suspicious destinations
- Mobile companion focused on screenshots and caregiver handoff
- Caseworker or community-advocate dashboard with explicit consent and retention controls
- Organization-level deployment, training, audit, and governance tooling

## Release Gates

Every phase must preserve the defensive purpose, local-first default, non-accusatory language, official-channel verification, sensitive-data minimization, accessibility, and independent security/privacy review.
